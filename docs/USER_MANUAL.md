## 1. システム概要

本システムは、外構・エクステリア工事業者向けのクラウド見積もり管理プラットフォームです。

### 主な機能

| 機能 | 説明 |
|------|------|
| 見積作成・管理 | 4階層の見積明細、PDF出力、バージョン管理 |
| 顧客管理 | 顧客情報の登録・管理、見積との紐付け |
| 単価マスタ | 頻繁に使う品名・単価のデータベース |
| メーカーカタログ | メーカー製品の定価カタログ管理 |
| テンプレート | よく使う見積パターンのテンプレート化 |
| 顧客共有ポータル | 顧客にURLを共有し、オンラインで承認を受ける |
| AI見積生成 | テキスト説明から見積明細を自動生成 |
| AI受注予測 | 過去データに基づく受注確率予測 |
| 経営分析 | 月別トレンド、カテゴリ別・顧客別・担当者別分析 |
| 通知システム | メール・LINEでの見積ステータス通知 |

## 2. ログイン・初期設定

### ログイン
1. ブラウザで `http://localhost:3000` にアクセス
2. メールアドレスとパスワードを入力してログイン
3. 初回ログイン時は会社情報とユーザー名を設定

### 開発モード
- `DEV_BYPASS_AUTH=true` 設定時は認証なしで自動ログイン
- サンプル会社「サンプルエクステリア株式会社」が自動作成されます

## 3. ダッシュボード

**URL:** `/dashboard`

### 画面構成

#### KPIカード（画面上部）
今月の実績を4つのカードで表示します。

| カード | 内容 |
|--------|------|
| 今月見積数 | 今月作成した見積の件数 |
| 今月見積総額 | 今月の見積金額合計 |
| 今月受注率 | 今月の受注率（受注数 ÷ 見積数） |
| 今月平均粗利率 | 今月の平均粗利率（オーナー・マネージャーのみ表示） |

#### 通知バナー
- 有効期限が7日以内の提出済み見積がある場合、黄色のバナーで件数を表示
- クリックすると該当の見積一覧にジャンプ

#### パイプライン
全期間のステータス別件数と金額を表示。各ステータスをクリックすると見積一覧にフィルター移動。

| ステータス | 意味 |
|-----------|------|
| 作成中 | 作成途中の見積 |
| 提出済 | 顧客に提出済み |
| 受注 | 顧客が承認した見積 |
| 失注 | 顧客にお断りされた見積 |
| 期限切れ | 有効期限が過ぎた見積 |

#### 最近の見積
直近10件の見積を表示。クリックで詳細ページへ。

#### 経営分析ボタン
右上の「経営分析」ボタンで詳細な分析ページへ移動。

## 4. 見積管理

### 4.1 見積一覧

**URL:** `/estimates`

#### 検索・フィルター
- **テキスト検索**: 見積番号・件名・顧客名で検索
- **ステータスフィルター**: 上部のボタンで絞り込み（すべて/作成中/提出済/受注/失注/期限切れ）
- **詳細フィルター**: 「詳細フィルター」をクリックで展開
  - 見積日（開始〜終了）
  - 金額（下限〜上限）

#### ソート
テーブルヘッダーをクリックで昇順/降順を切り替え。対応列：
- 件名
- 合計金額
- 見積日
- 作成日

#### ページネーション
1ページ20件で表示。前へ/次へボタンで移動。

---

### 4.2 新規見積作成

**URL:** `/estimates/new`

#### 操作手順

1. 見積一覧の「新規見積」ボタンをクリック
2. 基本情報を入力

| 項目 | 必須 | 説明 |
|------|------|------|
| 件名 | ○ | 見積の件名（例：〇〇様邸 外構工事） |
| 顧客 | - | ドロップダウンから選択 |
| 現場住所 | - | 工事現場の住所 |
| 見積日 | - | 見積書の日付（デフォルト：本日） |
| 有効期限 | - | 見積の有効期限（会社設定のデフォルト日数で自動計算） |

3. 見積明細を入力

#### 明細の階層構造（4レベル）

```
レベル1: 工種（例：門まわり工事）
  └ レベル2: 大項目（例：門柱工事）
    └ レベル3: 中項目（例：本体）
      └ レベル4: 品名（例：機能門柱（アルミ）W400×H1600）← 数量・単価を入力
```

- 「+ 工種を追加」ボタンでレベル1を追加
- 各項目の「+」ボタンで子項目を追加
- レベル4の品名のみ、数量・単位・単価・原価を入力
- 金額は自動計算（数量 × 単価）

#### テンプレートからの作成
1. 画面上部の「テンプレートから作成」ボタンをクリック
2. テンプレートを選択（システム/共有/個人から選べます）
3. テンプレートの明細が自動入力されます

#### AI自動生成
1. 画面上部の「AI生成」ボタンをクリック
2. 工事内容を日本語で説明（例：「駐車場2台分、フェンス20m、アプローチ」）
3. 必要に応じて現場写真を添付
4. 「見積を生成」をクリック
5. AIが4階層の見積明細を自動生成
6. 生成結果を確認し、必要に応じて修正

※ AI生成にはANTHROPIC_API_KEYの設定が必要です

4. 金額設定

| 項目 | 説明 |
|------|------|
| 諸経費率 | 小計に対する諸経費の割合（%） |
| 値引き | 金額指定 または 割合指定 |
| 消費税率 | デフォルトは会社設定の税率 |

5. 備考・メモを入力

| 項目 | 説明 |
|------|------|
| 備考 | 見積書（PDF）に記載される備考 |
| 社内メモ | PDF非出力の社内向けメモ |
| 支払条件 | 支払い条件（例：契約時50%、完了時50%） |

6. 「保存」ボタンで保存

#### モバイル対応
- スマートフォンではカード型のドリルダウンUIが自動表示
- パンくずナビで階層を移動
- レベル4でインライン編集（規格・数量・単位・単価・原価）

---

### 4.3 見積詳細

**URL:** `/estimates/[id]`

#### 表示内容
- 見積番号・バージョン・ステータスバッジ
- 顧客情報・現場住所・見積日・有効期限
- 階層表示の見積明細
- 金額合計（小計・諸経費・値引き・消費税・合計）
- 原価合計・粗利（オーナー・マネージャーのみ）
- AI受注予測（作成中/提出済の見積のみ、オーナー・マネージャーのみ）
- バージョン履歴（複数バージョンがある場合）

#### アクションボタン

| ボタン | 機能 | 権限 |
|--------|------|------|
| 提出済にする / 受注 / 失注 | ステータスを変更 | 編集権限者 |
| 編集 | 見積を編集画面で開く | 編集権限者 |
| 新バージョン | 同じ見積番号で新バージョンを作成 | 編集権限者 |
| 複製 | 新しい見積番号でコピーを作成 | 編集権限者 |
| テンプレ保存 | 現在の見積をテンプレートとして保存 | オーナー/マネージャー |
| 顧客に共有 | 共有URLを生成してコピー | 編集権限者 |
| PDF | PDF形式でダウンロード | 全ユーザー |
| 削除 | 見積を削除（確認ダイアログあり） | オーナー/マネージャー |

#### ステータス遷移フロー

```
作成中 → 提出済 → 受注（最終）
              ↘ 失注 → 作成中に戻す
期限切れ → 作成中に戻す
```

#### バージョン管理
- 同じ見積番号で複数バージョンを管理
- 「新バージョン」ボタンでv2, v3...を作成
- バージョン履歴カードで全バージョンを一覧表示
- 各バージョンをクリックで詳細表示

#### 受注/失注の見積を編集する場合
- 警告バナーが表示されます：「この見積は「受注」ステータスです。編集すると内容が上書きされます。」

#### 同時編集の防止
- 楽観的ロックにより、他のユーザーが同時に編集した場合にエラーが表示されます
- 「他のユーザーが変更を保存しています。ページを再読み込みしてください。」

---

### 4.4 PDF出力

見積詳細ページまたは編集ページの「PDF」ボタンをクリックすると、ブラウザの新しいタブでPDFが表示されます。

#### PDF内容
- 会社ロゴ（設定されている場合）
- 会社情報（名称・住所・電話番号等）
- 見積番号・見積日・有効期限
- 顧客情報
- 見積明細（4階層の階層表示）
- 金額合計（小計・諸経費・値引き・消費税・合計）
- 備考・支払条件

#### 内部用PDF（原価表示）
URLに `?type=internal` を追加すると、原価が表示されたPDFを出力できます（オーナー・マネージャーのみ）。

## 6. 単価マスタ

**URL:** `/master/unit-prices`

### 6.1 概要
頻繁に使用する品名・単価を登録しておき、見積作成時に素早く入力するためのデータベースです。

### 6.2 検索・フィルター
- **テキスト検索**: 品名・規格・メーカーで検索
- **カテゴリフィルター**: ドロップダウンでカテゴリ絞り込み

### 6.3 単価の登録・編集

| 項目 | 必須 | 説明 |
|------|------|------|
| 品名 | ○ | 品名（例：アルミフェンス H800） |
| 規格 | - | 規格・仕様 |
| 単位 | ○ | 式, ㎡, m, m³, 個, 台, 本, 人工, セット |
| 見積単価 | ○ | 顧客に提示する単価 |
| 原価 | - | 仕入原価（見積単価以下であること） |
| カテゴリ | - | カテゴリ分類 |
| メーカー | - | メーカー名 |
| 型番 | - | 型番 |
| 備考 | - | メモ |

### 6.4 一括価格調整

1. ツールバーの「一括調整」ボタンをクリック
2. 設定項目：

| 項目 | 説明 |
|------|------|
| カテゴリ | 調整対象のカテゴリ（全カテゴリも可） |
| 対象 | 見積単価のみ / 原価のみ / 両方 |
| 調整率（%） | 正の値で値上げ、負の値で値下げ（-50%〜+100%） |

3. 「一括調整を実行」で適用
4. 変更は価格変動履歴に自動記録されます

### 6.5 CSVインポート・エクスポート

#### インポート（CSV → システム）
CSVの必須列: **品名**, **単位**, **見積単価**

```csv
品名,規格,単位,見積単価,原価,メーカー,型番
アルミフェンス H800,横格子タイプ,m,8500,6000,LIXIL,YS3-T8
```

#### エクスポート（システム → CSV）
ツールバーの「CSVエクスポート」ボタンで全データをダウンロード。

### 6.6 価格インテリジェンス
- 見積編集画面で、単価入力欄の横に色付きドットが表示されます
- 過去の見積実績に基づく価格ベンチマーク

| 色 | 意味 |
|----|------|
| 緑 | 適正（平均±10%以内） |
| 黄 | 注意（平均より10〜25%乖離） |
| 赤 | 要確認（平均より25%以上乖離） |

※ 同じ品名で2件以上の見積実績がある場合に表示されます

## 8. テンプレート

**URL:** `/master/templates`

### 8.1 概要
よく使う見積パターンをテンプレートとして保存し、新規見積作成時に一括適用できます。

### 8.2 テンプレートの種類

| 種別 | 説明 |
|------|------|
| システム | 全社共通のプリセットテンプレート（編集・削除不可） |
| 共有 | チーム全体で使えるテンプレート |
| 個人 | 作成者のみが使えるテンプレート |

### 8.3 システムテンプレート（20種）

初期設定で以下の外構工事テンプレートが利用可能です：

| カテゴリ | テンプレート名 |
|---------|--------------|
| 門まわり | 門柱・ポスト設置 / 門扉設置 |
| 塀・フェンス | アルミフェンス工事 / ブロック塀工事 |
| カーポート | カーポート1台用 / カーポート2台用 |
| テラス・デッキ | ウッドデッキ設置 / テラス屋根設置 |
| アプローチ | インターロッキング舗装 / タイル舗装アプローチ |
| 駐車場 | 駐車場1台分 / 駐車場2台分 |
| 植栽・造園 | シンボルツリー植栽 / 庭園造園一式 |
| 照明・電気 | ガーデンライト設置 / 門灯・表札灯設置 |
| 排水・給水 | 立水栓設置 / 排水工事一式 |
| 付帯工事 | 残土処分・整地 / 既存構造物撤去 |

各テンプレートには業界標準的な数量・単価が設定されています。

### 8.4 テンプレートの作成

#### 新規作成
1. 「新規テンプレート」ボタンをクリック
2. テンプレート名・説明を入力
3. 「チーム全体で共有する」をONにすると共有テンプレートになります
4. 明細項目を追加（工種/大項目/中項目/品名）
5. 「保存」をクリック

#### 見積からテンプレート作成
1. 見積詳細ページの「テンプレ保存」ボタンをクリック
2. テンプレート名と説明を入力
3. 「保存」をクリック
4. 見積の全明細がテンプレートとしてコピーされます

### 8.5 テンプレートの適用
1. 新規見積作成画面を開く
2. 「テンプレートから作成」ボタンをクリック
3. テンプレートを選択
4. 明細が一括で入力されます
5. 必要に応じて数量や単価を修正

## 9. 顧客向け共有ポータル

### 9.1 概要
見積を顧客にオンラインで共有し、承認・お断り・質問をブラウザ上で受け付ける機能です。顧客はログイン不要でアクセスできます。

### 9.2 共有手順

1. 見積詳細ページの「顧客に共有」ボタンをクリック
2. 共有URLが生成されます
3. URLをコピーして顧客にメール・LINEで送付

### 9.3 顧客側の操作（ログイン不要）

顧客がURLにアクセスすると以下が表示されます：

- 会社名・御見積書タイトル
- 見積情報（件名・見積番号・日付・有効期限）
- 見積明細（階層表示、金額表示）
- 合計金額
- 備考・支払条件

#### 顧客のアクション

| ボタン | 説明 |
|--------|------|
| 承認する | 見積を承認（ステータスが「受注」に変更） |
| お断りする | 見積をお断り（ステータスが「失注」に変更） |
| 質問・コメントする | 質問やコメントを送信 |

- 有効期限が切れている場合は「有効期限が切れています」と表示され、承認できません
- 承認/お断り時に、見積作成者にメール・LINE通知が送信されます（通知設定が有効な場合）

### 9.4 コメント機能
- 顧客と業者間でインラインのQ&Aが可能
- コメントは時系列で表示
- 顧客コメントは青背景、業者コメントは灰色背景

## 10. 経営分析

**URL:** `/dashboard/analytics`

ダッシュボードの「経営分析」ボタンからアクセス。

### 10.1 年間サマリーKPI
- 年間見積数（受注数・受注率）
- 年間見積総額
- 年間粗利（オーナー/マネージャーのみ）
- 平均粗利率（オーナー/マネージャーのみ）

### 10.2 グラフ（過去12ヶ月）

| グラフ | 内容 |

## 12. AI機能

### 12.1 AI見積自動生成

#### 必要な設定
`.env` ファイルに以下を追加：
```
ANTHROPIC_API_KEY=sk-ant-xxxxx
```

#### 使い方
1. 新規見積作成画面で「AI生成」ボタンをクリック
2. 工事内容をテキストで説明

例：
```
駐車場2台分のコンクリート打設、
アルミフェンス20m（H800）、
門柱・ポスト設置、
アプローチのインターロッキング舗装10㎡
```

3. 現場写真がある場合は添付（任意）
4. 「見積を生成」をクリック
5. 30秒ほどでAIが4階層の見積明細を自動生成
6. 自社の単価マスタに一致する品名がある場合は自社価格を使用
7. 生成結果を確認し、必要に応じて修正して保存

### 12.2 AI価格インテリジェンス

見積編集画面で、レベル4（品名）の単価入力欄の横に色付きドットが自動表示されます。

- 過去の見積実績（受注/提出済）から品名別の平均単価を分析
- 現在入力中の単価と比較して、適正/注意/要確認を判定
- ドットにマウスカーソルを合わせると詳細情報が表示されます

### 12.3 AI受注確率予測

見積詳細ページで、作成中/提出済の見積に対して受注確率を予測表示します（オーナー/マネージャーのみ）。

#### 分析要因
| 要因 | 内容 |
|------|------|
| 金額 | 受注した見積の平均金額との比較 |
| 粗利率 | 受注した見積の平均粗利率との比較 |
| 顧客実績 | その顧客との過去の受注率 |

#### 表示
- 受注確率（%）をカラーバーで表示
- 緑（60%以上）/ 黄（40-59%）/ 赤（39%以下）
- 各要因の影響度と説明を表示

※ 最低5件の受注/失注データが必要です

## 14. 外部サービス連携

### 14.1 Supabase（必須）
- **用途**: データベース、認証
- **設定**: `.env` ファイルに `DATABASE_URL`, `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY` を設定

### 14.2 Anthropic Claude API（任意）
- **用途**: AI見積自動生成
- **設定**: `.env` ファイルに `ANTHROPIC_API_KEY` を設定
- **料金**: Anthropic API の従量課金

### 14.3 Resend（任意）
- **用途**: メール通知
- **設定**: `.env` ファイルに `RESEND_API_KEY` と `EMAIL_FROM` を設定
- **料金**: 月100通まで無料

### 14.4 LINE Messaging API（任意）
- **用途**: LINE通知
- **設定**: `.env` ファイルに `LINE_CHANNEL_ACCESS_TOKEN` を設定
- **準備**:
  1. LINE Developersでチャネルを作成
  2. チャネルアクセストークンを取得
  3. 通知を受けるユーザーはLINE公式アカウントと友だち追加が必要

### 環境変数一覧

```env
# 必須
DATABASE_URL=postgresql://...
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbG...

# 開発用
DEV_BYPASS_AUTH=true

# AI機能（任意）
ANTHROPIC_API_KEY=sk-ant-...

# メール通知（任意）
RESEND_API_KEY=re_xxxxx
EMAIL_FROM=noreply@your-domain.com

# LINE通知（任意）
LINE_CHANNEL_ACCESS_TOKEN=xxxxx
```

## 見積番号の形式

見積番号は自動採番されます。

```
{会社略称}-{年月}-{連番}
例: SMP-202604-0001
```

- 会社略称: 会社設定の「略称」フィールド
- 年月: 見積作成時の年月（YYYYMM）
- 連番: 月ごとにリセットされる4桁の連番

## トラブルシューティング

### よくある問題

| 問題 | 対処法 |
|------|--------|
| ログインできない | DEV_BYPASS_AUTH=true が設定されているか確認 |
| PDF生成でエラー | フォントの読み込みに時間がかかる場合があります。再試行してください |
| AI生成が動かない | ANTHROPIC_API_KEYが.envに設定されているか確認 |
| メール通知が届かない | RESEND_API_KEYが設定されているか確認。設定ページで通知がONになっているか確認 |
| カタログページでエラー | 開発サーバーを再起動してください（`npm run dev`） |
